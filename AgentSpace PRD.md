

# **PRD: AgentSpace (跨应用视窗与意图路由器)**

## **1\. 产品概述 (Product Overview)**

AgentSpace 是一款 Windows 桌面空间规划与意图路由工具。它通过调用系统底层 API，将用户需要的多个软件（如浏览器、记事本、本地终端）的“局部核心交互区”裁剪并聚合到一个名为“沙漏 (Hourglass)”的对称容器中。在此基础上，通过 UI Automation 实现跨窗口的免复制粘贴数据流转，打造极致的心流工作台。

## **2\. 核心架构设计 (Architecture)**

* **对称式容器 (Symmetrical Containers)：** UI 呈现为上下分布的两个相同容器组件，中间由“控制中枢 (Nexus Bar)”连接。容器本身无业务逻辑，仅作为原生窗口的视觉挂载点。  
* **隐身斗篷渲染法 (Window Masking Engine)：** 这是核心技术点。利用 Windows 底层 API（如 SetWindowRgn），对目标原生窗口进行物理裁剪。  
  * **要求：** 被用户框选保留的 10% 区域在容器内保持 100% 的原生交互（输入法跟随、滚动、点击）。  
  * **要求：** 剩余的 90% 区域必须被设置为完全透明，且具有“鼠标穿透 (Hit-Test Ignore)”属性，不遮挡桌面其他软件。  
* **UI 自动化通信 (UI Automation Data Flow)：** 基于 UIAutomationClient 实现无需剪贴板的文本流转。

## **3\. 核心功能模块与验收标准 (Features & Acceptance Criteria)**

### **3.1 极简截取与映射 (Pin & Map)**

* **功能描述：** 用户触发全局快捷键后，屏幕出现类似截图工具的蒙版。用户框选目标软件区域。  
* **技术实现与 AC：**  
  * 触发蒙版时，系统冻结当前屏幕画面供用户框选。  
  * 获取框选区域下方最顶层的原生窗口句柄 (HWND)。  
  * 计算框选区域相对于该窗口的相对坐标 (Rect)。  
  * **核心动作：** 将该窗口挂载到 AgentSpace 容器中，并根据计算出的 Rect 执行窗口遮罩 (SetWindowRgn)。

### **3.2 容器自适应与还原 (Container Scaling & Restore)**

* **功能描述：** 容器支持缩放与二次裁剪，且能无损还原。  
* **技术实现与 AC：**  
  * 拖拽容器边缘：等比例缩放挂载的原生窗口。  
  * 按住 Ctrl \+ 拖拽边缘：动态更新 SetWindowRgn 的区域参数，实现重新裁剪。  
  * 点击【还原】按钮：解除该窗口的遮罩状态，恢复其原始尺寸和屏幕坐标，并从容器中解绑。

### **3.3 意图驱动的数据流转 (Data Flow & Intent Routing)**

* **功能描述：** 在不同容器挂载的软件间实现文本的一键发送。  
* **技术实现与 AC：**  
  * **源获取 (Source)：** 当按下【发送】快捷键时，通过 UI Automation 获取当前激活容器中选中状态的文本内容。如果无法通过 UIA 获取，可降级使用后台模拟 Ctrl+C 提取。  
  * **目标注入 (Target)：** 焦点静默切换至由【切换目标】快捷键选定的目标容器。通过 UI Automation 将文本注入到目标软件的输入框中。  
  * **UI 反馈：** 准备流转时，源容器边框高亮（如蓝色），目标容器边框高亮（如绿色）。中枢横条 (Nexus Bar) 显示流转方向动画。

### **3.4 快捷键与按键映射管理器 (Hotkey Manager)**

* **功能描述：** 全局快捷键的监听与管理。  
* **技术实现与 AC：**  
  * **绝对规则：** 严禁默认覆盖操作系统核心快捷键（如 Alt+Tab, Win+D, Ctrl+C 等）。  
  * 【呼出截取蒙版】：默认绑定为 Ctrl+Shift+Space（可自定义）。  
  * 【切换目标容器】：**默认未绑定 (Unbound)**，提供 UI 供用户自定义配置。  
  * 【执行数据发送】：**默认未绑定 (Unbound)**，提供 UI 供用户自定义配置。  
  * 快捷键系统需使用系统级全局钩子 (Global Keyboard Hook，如 RegisterHotKey) 实现，并确保在软件退出时正确释放资源。

## **4\. MVP 阶段划分建议 (Phase 1 Scope)**

为了控制开发风险，请严格按照以下阶段进行开发：

* **Phase 1：** 跑通底层逻辑。实现全屏半透明蒙版、框选区域、获取底层 HWND，并成功调用 SetWindowRgn 让该窗口只有被框选部分可见（暂不涉及移动和 UI 自动化通信）。  
* **Phase 2：** 构建 AgentSpace 的 WPF 容器 UI，将被遮罩的窗口与 WPF 容器实现视觉联动（拖拽容器时，底层的残缺窗口跟着移动）。  
* **Phase 3：** 引入 UI Automation，完成跨窗口的文本提取与注入。
